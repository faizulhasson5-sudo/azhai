Technical SEO Checklist 2026

A complete audit checklist to ensure search engines can crawl, render, and index your website effectively. Use this step-by-step guide to identify and fix technical SEO issues.

1. Crawlability

Crawlability refers to search engines' ability to access and crawl your pages. If crawlers cannot reach your content, it will never appear in search results.

Robots.txt

  • Verify robots.txt exists at yourdomain.com/robots.txt
  • Ensure important pages are not blocked by Disallow rules
  • Allow CSS, JavaScript, and image directories
  • Include Sitemap directive pointing to your XML sitemap
  • Do not use robots.txt to prevent indexing (use noindex instead)

XML Sitemap

  • Sitemap exists at yourdomain.com/sitemap.xml
  • Includes all important pages (exclude tag pages, thin content, noindex pages)
  • Uses absolute URLs with https protocol
  • Contains accurate lastmod dates
  • Under 50,000 URLs and 50MB uncompressed
  • Submitted to Google Search Console and Bing Webmaster Tools
  • Referenced in robots.txt

Crawl Errors

  • Check Google Search Console Coverage report for errors
  • Fix 404 errors (redirect or restore removed pages)
  • Fix 500 server errors (check server logs)
  • Fix soft 404 errors (pages returning 200 but showing "not found" content)
  • Fix redirect chains (max 1 redirect hop)
  • Fix redirect loops (infinite redirect chains)

2. Indexation

After crawling, search engines must index your pages to include them in search results.

  • Check indexing status in Google Search Console URL Inspection tool
  • Ensure important pages are indexed (site:yourdomain.com in Google)
  • Add noindex tags to pages that should not appear in search results
  • Fix "Discovered - currently not indexed" status (improve content quality or internal linking)
  • Fix "Crawled - currently not indexed" status (pages crawled but not deemed worthy of indexing)
  • Use canonical tags to consolidate duplicate content

3. Site Speed and Core Web Vitals

Page speed is a confirmed ranking factor. Google's Core Web Vitals measure real-world user experience.

Largest Contentful Paint (LCP)

  • Target: under 2.5 seconds
  • Optimize images (compress, use WebP format, lazy load)
  • Use a CDN for static assets
  • Minimize render-blocking resources
  • Preload critical resources

Interaction to Next Paint (INP)

  • Target: under 200 milliseconds
  • Minimize JavaScript execution time
  • Break up long tasks
  • Use web workers for heavy computations

Cumulative Layout Shift (CLS)

  • Target: under 0.1
  • Set explicit width and height for images and videos
  • Avoid inserting content above existing content
  • Use CSS aspect-ratio for responsive media

4. Mobile-Friendliness

Google uses mobile-first indexing. Your mobile site is the primary version for indexing and ranking.

  • Site uses responsive design (same URL for mobile and desktop)
  • Text is readable without zooming (16px minimum font size)
  • Buttons and links are easily tappable (48x48px minimum tap target)
  • Content fits the screen without horizontal scrolling
  • No intrusive interstitials (pop-ups) on mobile
  • Viewport meta tag is properly configured

5. HTTPS and Security

HTTPS is a confirmed ranking factor. All pages should use HTTPS.

  • SSL certificate is valid and not expired
  • All pages load over HTTPS (no mixed content warnings)
  • HTTP pages redirect to HTTPS (301 redirect)
  • HSTS header is configured (optional but recommended)

6. Site Architecture

Good site architecture helps both users and search engines navigate your content.

  • Flat architecture: important pages within 3 clicks of homepage
  • Logical URL hierarchy: domain.com/category/page
  • Internal linking connects related content
  • Breadcrumb navigation on all pages
  • No orphan pages (pages not linked from anywhere)
  • HTML sitemap for users (optional)

7. URL Structure

  • Short, descriptive URLs (under 75 characters)
  • Include target keyword in URL
  • Use hyphens to separate words
  • Avoid parameters, session IDs, and unnecessary strings
  • Consistent URL structure across the site
  • No uppercase letters in URLs

8. Structured Data

Structured data helps search engines understand your content and can generate rich results.

  • Implement Organization schema on homepage
  • Implement BreadcrumbList schema on all pages
  • Implement Article schema on blog posts
  • Implement FAQPage schema on pages with FAQ sections
  • Implement Product schema on product pages
  • Validate all schema with Google Rich Results Test
  • Monitor for schema errors in Google Search Console

9. International SEO

If your site targets multiple countries or languages:

  • Implement hreflang tags for each language/country version
  • Use correct language and country codes (en-US, en-GB)
  • Each page should reference all language versions including itself
  • Use a CDN with edge locations in target countries
  • Consider country-specific domain extensions for major markets

10. Log File Analysis

Server log files show how search engine crawlers actually access your site.

  • Check crawl frequency (how often Googlebot visits)
  • Identify pages with high crawl rates (popular with crawlers)
  • Identify pages with low or no crawl rates (potentially overlooked)
  • Detect crawl budget waste on low-value pages
  • Check for 404 errors and redirect chains in crawl logs
